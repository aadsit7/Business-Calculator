Benchmark starting point

Defaults are designed to be credible in a first conversation and easy to replace with account data.

Labor and packaging

  • $70/hour loaded labor: rounded blended U.S. starting point using BLS pay for systems administrators and support specialists plus the BLS private-industry benefit share.
  • 8 hours per current package: conservative versus a 2025 study reporting $646 average cost for a Windows application package.
  • 1.5 / 4 hours with AW: residual standard/custom effort retained for intake, testing, approvals, exceptions, and monitoring.

Tickets and realization

  • 0.20 application tickets per user/month: below the cross-source implication of 1.1 total agent-assisted tickets and a 38.2% Software & Applications share.
  • 20-minute handling time: includes analyst work, documentation, follow-up, and closure.
  • 25% ticket reduction: aligned with MetricNet’s description of excellent annual progress, then reduced again by the 75% realization factor.

Core formulas

All economics are driven by entered workload.

Current package hours = packages changed per month x current hours per package AW target package hours = packages changed per month x [(standard share x AW standard hours) + (custom share x AW custom hours)] Effective AW package hours = current package hours - [(current package hours - AW target package hours) x realization factor] Effective AW patch hours = current patch hours x [1 - (patch reduction x realization factor)] Effective AW ticket hours = current ticket hours x [1 - (ticket reduction x realization factor)] Steady-state AW cost = AW labor hours x labor cost + annual license + other annual AW operating cost Year-one savings ramp = linear from zero to steady state during entered ramp months MSP margin recovery = current operating cost - AW steady-state operating cost SI date movement = workstream weeks saved x critical-path capture

Product facts used in the model

Platform scope and licensing boundaries that affect the inputs.

Supported positioning

  • Application Workspace complements Intune, Configuration Manager, Citrix, AVD, Windows 365, and other existing investments; it is not a rip-and-replace endpoint or VDI platform.
  • It provides a Workspace and Catalog experience for entitled applications and resources.
  • Packages can include distribute, install, launch, uninstall, repair, filter, dependency, and entitlement logic.
  • Managed packages can receive new versions through the Setup Store connector.
  • Multiple zones can be used for separate clients; each zone requires licensing.

Licensing boundaries

  • Release & Patch Management is a separate add-on and is required for full Setup Store connector capability and release-stage functions.
  • Access Manager is a separate add-on for integrated single sign-on capabilities.
  • Managed updates still require the partner or customer to define testing, approval, scheduling, and rollout policy.
  • Catalog eligibility varies; internal, legacy, and highly customized apps retain more manual work.
